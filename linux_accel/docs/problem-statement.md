## 背景

eBPF 可以在 Linux 内核中安全运行沙盒程序，不需要修改内核源码，也不需要加载传统内核模块。

对于网络服务，eBPF 可用于：

- 实时监控服务性能；
- 在内核态处理部分服务请求；
- 实现灵活缓存策略；
- 降低主机网卡与虚拟机内应用之间的通信开销；
- 优化虚拟化场景中的数据包路径。

## 任务要求

- 使用 eBPF 实时监控服务性能。
- 使用 eBPF 实现性能优化，例如优化 host-to-VM 通信开销或实现双端缓存。
- 支持至少两种网络服务，例如 gRPC、DNS、LDAP。

## 预期特征

- 对至少两种服务实现 eBPF 监控和加速。
- 优化 KVM 等虚拟化场景中的数据包处理流程。
- 设计支持动态配置客户端侧与服务端侧策略的双端缓存机制。
- 通过 benchmark 数据展示性能提升。
